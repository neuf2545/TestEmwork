# ข้อ 4: ManufacturePlus Scenario

## 1. Functional Requirements 3 ข้อที่ช่วยลดการเดินมาถาม HR เรื่องวันลา
เพื่อแก้ปัญหาคอขวดที่ HR ระบบใหม่จะต้องมีฟีเจอร์แบบ **Employee Self-Service (ESS)** เพื่อให้พนักงานจัดการตัวเองได้ ดังนี้:

1. **ระบบแสดงยอดวันลาคงเหลือแบบเรียลไทม์ (Real-time Leave Balance Dashboard):**
   - ระบบต้องมีหน้าจอ (ผ่าน Web Portal หรือ Mobile App) ที่พนักงานสามารถล็อกอินเข้ามาดูโควตาวันลาทุกประเภท (ลาพักร้อน, ลาป่วย, ลากิจ) รวมถึงประวัติการลาที่ผ่านมาได้ด้วยตัวเองตลอด 24 ชม. โดยตัวเลขต้องหักลบอัตโนมัติทันทีที่มีการอนุมัติ
2. **ระบบยื่นคำขอและอนุมัติออนไลน์ (Online Leave Request & Workflow):**
   - ระบบต้องอนุญาตให้พนักงานกดยื่นลางานผ่านแอปพลิเคชัน โดยระบบจะทำการตรวจสอบโควตาก่อน หากโควตาพอ ระบบจะส่งคำขอไปยังหัวหน้างาน (Line Manager) เพื่อพิจารณาอนุมัติผ่านระบบดิจิทัล (ลดการใช้กระดาษและลัดขั้นตอนที่ไม่ต้องผ่าน HR)
3. **ระบบแจ้งเตือนสถานะอัตโนมัติ (Automated Notification System):**
   - ระบบต้องสามารถส่งการแจ้งเตือน (เช่น ผ่าน LINE OA, Email หรือ Push Notification) ไปยังพนักงานทันทีเมื่อคำขอลาได้รับการ "อนุมัติ" หรือ "ปฏิเสธ" รวมถึงสามารถตั้งค่าส่งสรุปยอดวันลาคงเหลือประจำเดือนให้พนักงานทราบอัตโนมัติ

## 2. เทคนิค Data Cleansing (การทำความสะอาดข้อมูลก่อนนำเข้าระบบใหม่)
การใช้ Excel มา 10 ปีกับพนักงาน 800 คน จะมีปัญหา Data Quality แน่นอน เทคนิคในการ Cleansing ข้อมูลมีดังนี้:

### A. การจัดการปัญหา Employee ID ซ้ำกัน (Handling Duplicates)
- **Data Profiling & Isolate:** ใช้เครื่องมือวิเคราะห์ข้อมูลเบื้องต้น (เช่น Pivot Table, `COUNTIF` ใน Excel หรือนำเข้า Staging Database แล้วใช้ SQL `GROUP BY ... HAVING COUNT > 1`) เพื่อกางข้อมูลที่ ID ซ้ำกันออกมาทั้งหมด
- **Cross-Referencing (ตรวจสอบไขว้):** นำ ID ที่ซ้ำมาตรวจสอบเทียบกับข้อมูลที่เป็น **"Unique Identifier ตัวจริง"** เช่น เลขบัตรประจำตัวประชาชน (National ID) หรือเลขที่บัญชีธนาคาร
  - *ถ้าเป็นคนเดียวกัน (Row ซ้ำซ้อนจากการ Copy-Paste):* ให้ใช้เทคนิค **Deduplication** คือยุบรวมข้อมูล (Merge) และเลือกเก็บเฉพาะแถวที่มีข้อมูลล่าสุด (Latest Update) ไว้
  - *ถ้าเป็นคนละคน (HR พิมพ์รหัสผิด):* ให้สร้าง Employee ID ใหม่ตามฟอร์แมตมาตรฐานของบริษัทให้กับพนักงานที่รหัสผิด แล้วนำไปอัปเดตในระบบ

### B. การจัดการปัญหาชื่อพนักงานพิมพ์ผิด หรือฟอร์แมตไม่ตรงกัน (Handling Misspelled Names)
- **Data Standardization (จัดระเบียบรูปแบบคำ):** ใช้กระบวนการลบช่องว่างส่วนเกิน ลบตัวอักษรพิเศษที่ซ่อนอยู่ (เช่น ใช้คำสั่ง `TRIM()`, `CLEAN()`) และปรับรูปแบบตัวอักษรให้อยู่ในมาตรฐานเดียวกัน
- **Fuzzy Matching & Clustering:** ใช้เครื่องมือ Data Cleansing (เช่น OpenRefine หรือ Python) เพื่อหาความคล้ายคลึงของตัวอักษร (เช่น "สมศักดิ์" กับ "สมศ้กดิ์") ระบบจะช่วยจัดกลุ่มชื่อที่พิมพ์ผิดให้มารวมกัน เพื่อให้ Data Analyst กดแก้ไขรวดเดียว (Mass Edit) ได้ง่ายขึ้น
- **Master Data Validation (เทียบข้อมูลจากแหล่งที่ถูกต้องที่สุด):** หากไม่แน่ใจว่าชื่อไหนสะกดถูกต้อง ให้ทำการ Mapping เทียบกับไฟล์ส่งประกันสังคม (SSO) สรรพากร (ภ.ง.ด.) หรือไฟล์จ่ายเงินเดือนผ่านธนาคาร (Bank Payroll) เนื่องจากเอกสารเหล่านี้มีการยืนยันตัวตนทางกฎหมายแล้ว มักจะเป็นชื่อที่สะกดถูกต้องที่สุด

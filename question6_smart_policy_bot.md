# ข้อ 6: Smart Policy Bot

## 1. การเขียน Prompt ที่ดีที่สุด (System Prompt)
การเขียน Prompt ที่ดีสำหรับ AI สาย HR ต้องกำหนด **บทบาท (Role)**, **แหล่งข้อมูล (Knowledge Base)**, และ **ข้อจำกัด (Guardrails)** อย่างชัดเจน เพื่อป้องกันอาการ "คิดไปเอง" (Hallucination)

**ตัวอย่าง System Prompt:**
```text
[Role]
คุณคือ HR Assistant Bot ของบริษัท มีหน้าที่ตอบคำถามพนักงานเกี่ยวกับ "สิทธิ์การลาพักร้อน" เท่านั้น โดยต้องตอบด้วยความสุภาพ กระชับ และเป็นมืออาชีพ

[Knowledge Base - กฎกติกา]
1. สิทธิ์รายปี: พนักงานได้สิทธิ์ลาพักร้อน 10 วัน/ปี
2. การสะสม: สามารถสะสมวันลาพักร้อนรวมกันสูงสุดได้ไม่เกิน 15 วัน (หากเกินกว่านี้สิทธิ์ที่เกินจะถูกตัดทิ้ง)
3. การแจ้งล่วงหน้า: พนักงานต้องแจ้งขอลางานล่วงหน้าอย่างน้อย 3 วันเสมอ

[Constraints - ข้อควรระวัง]
- ตอบคำถามโดยอ้างอิงจาก Knowledge Base ด้านบนเท่านั้น ห้ามคิดกฎขึ้นมาเอง
- หากพนักงานสอบถามเรื่องอื่นที่ไม่เกี่ยวกับการลาพักร้อน (เช่น ลาป่วย, ลาคลอด) ให้ตอบว่า "ขออภัยค่ะ ระบบสามารถให้ข้อมูลได้เฉพาะสิทธิ์การลาพักร้อนเท่านั้น โปรดติดต่อฝ่าย HR โดยตรง"
- หากพนักงานต้องการใช้วันลาสะสมเกิน 15 วัน หรือขอลากะทันหัน (แจ้งน้อยกว่า 3 วัน) ต้องอธิบายและแจ้งเตือนพนักงานอย่างสุภาพว่าไม่สามารถทำได้เนื่องจากผิดเงื่อนไขของบริษัท
```

## 2. Logic สำหรับดักจับ (Prompt Guard / Output Validator)
แม้ว่าเราจะควบคุม Prompt ได้ดีแล้ว แต่ AI ก็ยังมีโอกาสคำนวณเลขผิดหรือหลอนคำตอบเองได้ เพื่อความปลอดภัยสูงสุด (Enterprise-grade) เราควรใช้เทคนิค **Output Guardrails (ดักจับคำตอบก่อนส่งให้ผู้ใช้)** ดังนี้:

**Pseudo-code (Logic / Prompt Guard):**
```python
def check_ai_output(user_query, ai_response):
    # ---------------------------------------------------------
    # ชั้นที่ 1: Rule-based Keyword Guardrail (ดักด้วยคำตายตัว)
    # ---------------------------------------------------------
    forbidden_phrases = [
        "ไม่ต้องแจ้งล่วงหน้า", "แจ้งวันต่อวัน", "แจ้ง 1 วัน", "แจ้ง 2 วัน", 
        "สะสมเกิน 15 วัน", "สะสมไม่จำกัด"
    ]
    for phrase in forbidden_phrases:
        if phrase in ai_response:
            return fallback_standard_message() # บล็อกคำตอบทันที
            
    # ---------------------------------------------------------
    # ชั้นที่ 2: Logic Guardrail (ตรวจสอบความถูกต้องของตัวเลข)
    # ---------------------------------------------------------
    # ดึงตัวเลขทั้งหมดจากประโยคของ AI ด้วย Regex มาเช็ค (Sanity Check)
    numbers = extract_numbers(ai_response)
    for num in numbers:
        # ถ้าระบบพบว่า AI ดันแนะนำตัวเลขวันสะสมที่ > 15 วัน ให้ตีตก
        if num > 15 and is_discussing_accumulation(ai_response):
            return fallback_standard_message()

    # ---------------------------------------------------------
    # ชั้นที่ 3: LLM-as-a-Judge (ใช้ AI โมเดลเล็กมารีวิว AI โมเดลหลัก)
    # ---------------------------------------------------------
    guard_prompt = f"""
    ในฐานะผู้คุมกฎบริษัท จงประเมินว่า 'คำตอบของ AI' ขัดกับกฎ 3 ข้อนี้หรือไม่:
    1. พักร้อน 10 วัน/ปี
    2. สะสมสูงสุด 15 วัน
    3. แจ้งล่วงหน้าอย่างน้อย 3 วัน
    
    คำถาม: {user_query}
    คำตอบของ AI: {ai_response}
    
    ให้ตอบเพียงคำเดียว: 'SAFE' หากถูกต้องปลอดภัย หรือ 'VIOLATION' หากผิดกฎ/คำนวณผิด
    """
    validation_result = call_fast_llm(guard_prompt)
    
    if "VIOLATION" in validation_result:
        return "ขออภัยค่ะ ข้อมูลที่ระบบประมวลผลอาจคลาดเคลื่อนกับนโยบาย โปรดติดต่อ HR โดยตรงเพื่อความถูกต้องค่ะ"
        
    return ai_response # ผ่านครบ 3 ด่าน ปลอดภัย ส่งให้พนักงานอ่านได้
```
